# Wokku Marketing Content Pack

> Everything you need for Wokku's soft launch marketing. Text-based, copy-paste ready, no tools required.

## Files

1. **[01-content-scripts-month-1.md](01-content-scripts-month-1.md)** — 30 video scripts for TikTok/Reels, 60 seconds each, all in Bahasa Indonesia
2. **[02-hashtags-and-captions.md](02-hashtags-and-captions.md)** — Platform-specific captions, hashtag strategy, posting schedule
3. **[03-response-templates.md](03-response-templates.md)** — Copy-paste replies for DMs, comments, common questions
4. **[04-ad-copy-variants.md](04-ad-copy-variants.md)** — 10+ paid ad variants for Meta/Google/TikTok/LinkedIn

## How to Use This Pack

### Week 1: Setup (do before posting anything)
- [ ] Create TikTok, Instagram Reels, Twitter, LinkedIn accounts (or use existing)
- [ ] Set up Buffer free tier (3 channels, 10 scheduled posts)
- [ ] Buy a lapel mic (Rp 100-200rb)
- [ ] Download CapCut on phone
- [ ] Set up Sentry (error tracking) — critical before real users arrive
- [ ] Set up Plausible or Umami analytics on wokku.dev
- [ ] Add UTM tracking to all outbound links

### Week 2: Batch record first 10 videos
- [ ] Pick 10 scripts from `01-content-scripts-month-1.md` (Scripts 1-10 recommended)
- [ ] Record all 10 in 1-2 sessions (2-3 hours total)
- [ ] Edit in CapCut (add captions, music, transitions)
- [ ] Schedule in Buffer following the posting schedule

### Week 3-4: Publish + engage
- [ ] 3 videos/week on TikTok + Reels
- [ ] 1 LinkedIn post/week
- [ ] 1 Twitter thread/week
- [ ] Reply to every comment and DM within 12 hours
- [ ] Use response templates from `03-response-templates.md`

### Month 2: Scale what works
- [ ] Review analytics: which videos got most views?
- [ ] Double down on the format/topic that worked
- [ ] Start paid ads (budget Rp 2-5 juta) using `04-ad-copy-variants.md`
- [ ] Record 10 more scripts

## Minimum Viable Launch Checklist

Before you post your first video, make sure:

**Product:**
- [ ] wokku.dev loads fast
- [ ] Signup flow works end-to-end
- [ ] Free tier activates immediately
- [ ] Payment flow tested (QRIS works)
- [ ] At least 3 popular templates deploy successfully
- [ ] Support email (support@wokku.dev) set up and monitored

**Monitoring:**
- [ ] Sentry or Honeybadger tracking errors
- [ ] Plausible/Umami tracking pageviews
- [ ] Uptime monitoring (use your own Uptime Kuma template!)
- [ ] Slack/Telegram alerts for errors

**Content:**
- [ ] 10 videos recorded and scheduled
- [ ] Landing page ready (wokku.dev/)
- [ ] Documentation accessible from nav
- [ ] Pricing page clear
- [ ] FAQ answers the top 10 questions

**Support:**
- [ ] Telegram group created ("Wokku Indonesia")
- [ ] Discord server (optional, secondary)
- [ ] Response templates ready to copy-paste
- [ ] Founder (you) available to respond to DMs daily

## Red Flags to Watch For

1. **Crickets after posting** — if your first 3 videos get < 500 views each, you have a distribution problem. Fix before making more content.

2. **High bounce rate** — if people visit wokku.dev and leave in < 10 seconds, your landing page isn't clear enough.

3. **Signups but no deploys** — people register but never deploy an app. Your onboarding is broken.

4. **Deploys but no payment** — people deploy on free tier but never upgrade. Your value prop for paid tier isn't clear.

5. **Support overwhelm** — if you're spending > 4 hours/day answering DMs, you need to pause marketing and build self-serve docs.

## Success Metrics (be realistic)

**Month 1 (soft launch):**
- 50-200 signups
- 10-30 paying users
- 5-20 apps deployed per day
- $15-75 MRR

**Month 3:**
- 200-500 signups
- 50-100 paying users
- $75-300 MRR
- 1-2 user testimonials to use in future marketing

**Month 6:**
- 500-2,000 signups
- 150-400 paying users
- $300-1,200 MRR
- Product-market fit signals (high retention, organic referrals)

**Month 12:**
- 2,000-10,000 signups
- 500-2,000 paying users
- $1,500-6,000 MRR
- Sustainable bootstrapped business

These are realistic solo-founder targets. If you outperform, great. If you underperform, the content pack needs iteration.
